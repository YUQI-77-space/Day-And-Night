# 美术风格指南

## 整体风格定位

### 核心美术风格

**像素艺术（Pixel Art）**
- 分辨率：16x16 / 32x32 / 64x64（根据对象大小）
- 风格：现代像素风，细节丰富
- 参考：《星露谷物语》+《暗黑地牢》+《死亡细胞》

**视觉特点：**
- 清晰可读的像素线条
- 有限但精心选择的调色板
- 流畅的动画表现
- 末日氛围但不过度压抑
- 温暖与冷酷的对比

**情感基调：**
- 小镇内：温暖、希望、生活气息
- 外围区域：冷酷、危险、压抑
- 夜晚：神秘、紧张、恐怖
- 整体：末日中的人性光辉

## 色彩设计

### 主色调系统

#### 小镇色调（温暖）

**主色调：**
- 暖黄色：#F4A460（建筑、灯光）
- 棕色：#8B4513（木材、土地）

**辅助色：**
- 橙色：#FF8C00（火光、夕阳）
- 浅蓝：#87CEEB（天空、水）
- 红色：#CD5C5C（点缀、警示）

**使用场景：**
- 白天的小镇
- 室内温暖场景
- NPC服装
- UI界面

**主色调：**
- 灰色：#696969（废墟、混凝土）

**辅助色：**
- 紫色：#483D8B（感染、异常）
- 血红：#8B0000（危险、血迹）

**使用场景：**
- 废弃区域
- 夜晚场景
- 敌人设计
- 危险提示

### 昼夜色彩变化

**白天（6:00-18:00）：**
- 明亮的自然光
- 高饱和度
- 清晰的阴影
- 色温：暖色

**黄昏（18:00-19:00）：**
- 金色光芒
- 长阴影

- 月光和人造光

## 角色设计

### 玩家角色

**基础设计：**
- 尺寸：32x32 像素
- 风格：可辨识但不过度细节
- 可自定义：发型、肤色、服装

**动画帧数：**
- 待机：4帧
- 行走：6帧
- 跑步：6帧
- 攻击：4-6帧
- 闪避：4帧
- 受伤：2帧
- 死亡：6帧

**方向：**
- 四方向：上、下、左、右
- 或八方向（如果需要更精细）

**装备显示：**
- 武器：手持显示
- 防具：改变身体颜色/纹理
- 配饰：额外图层

### NPC设计

**核心NPC：**
- 尺寸：32x32 像素
- 独特的外观特征
- 丰富的表情变化
- 完整的动画集

**重要NPC：**
- 尺寸：32x32 像素
- 有特色但相对简单
- 基础动画（待机、行走）

**普通NPC：**
- 尺寸：32x32 像素
- 模块化设计（可复用）
- 最少动画

**NPC特征差异化：**
- 体型：高矮胖瘦
- 服装：职业特征明显
- 配色：个性化色彩
- 配饰：帽子、眼镜、工具等

### 敌人设计

#### 感染者类型

**普通感染者：**
- 尺寸：32x32 像素
- 特征：破烂衣服、苍白皮肤、血迹
- 动画：缓慢移动、攻击、死亡
- 颜色：灰白、暗红

**快速感染者：**
- 尺寸：32x32 像素
- 特征：瘦削、弯腰、利爪
- 动画：快速移动、扑击
- 颜色：深灰、紫色调

**重型感染者：**
- 尺寸：48x48 像素
- 特征：肌肉膨胀、畸形
- 动画：沉重移动、重击
- 颜色：暗红、黑色

**变异感染者：**
- 尺寸：64x64 像素（Boss）
- 特征：明显变异、发光部位
- 动画：复杂攻击模式
- 颜色：紫色、绿色荧光

**设计原则：**
- 威胁感：通过姿态和颜色传达
- 可辨识：远距离也能识别类型
- 动画流畅：攻击前摇明显
- 死亡效果：满足感的反馈

## 场景设计

### 瓦片集（Tileset）设计

**地面瓦片：**
- 尺寸：16x16 像素
- 类型：
  - 泥土路
  - 石板路
  - 草地
  - 木地板
  - 混凝土
  - 废墟地面

**墙壁瓦片：**
- 尺寸：16x16 像素
- 类型：
  - 木墙
  - 石墙
  - 砖墙
  - 金属墙
  - 破损墙壁

**装饰瓦片：**
- 尺寸：16x16 或更大
- 类型：
  - 家具
  - 植物
  - 路灯
  - 标识
  - 废弃物

**瓦片规则：**
- 支持自动拼接（Auto-tiling）
- 边缘和角落变体
- 随机变体（增加自然感）

### 建筑设计

**小镇建筑：**
- 风格：末日前的小镇建筑，有修补痕迹
- 特点：
  - 木质结构为主
  - 有防御加固
  - 窗户有木板封堵
  - 屋顶有太阳能板或风车

**废弃建筑：**
- 风格：破败、危险
- 特点：
  - 破损的墙壁
  - 倒塌的结构
  - 生锈的金属
  - 野草丛生

**室内场景：**
- 透视：俯视角（Top-down）
- 细节：家具、装饰、生活痕迹
- 光照：窗户光、灯光

### 环境元素

**自然元素：**
- 树木：多种形态，有枯萎和健康的对比
- 草丛：可互动，可隐藏
- 水体：河流、池塘，有倒影效果
- 天气：雨、雪、雾的视觉效果

**人造元素：**
- 路障：木栅栏、铁丝网
- 陷阱：可见的危险提示
- 标识：路牌、警告牌
- 废弃物：车辆残骸、垃圾堆

## UI设计

### UI风格

**整体风格：**
- 像素风格与UI结合
- 清晰易读
- 末日主题但不影响可用性
- 半透明背景，不遮挡游戏画面

**配色方案：**
- 主色：深灰 #2C2C2C
- 强调色：橙黄 #FFA500

## 动画设计

### 角色动画

**动画原则：**
- 流畅自然
- 有预备动作（Anticipation）
- 有跟随动作（Follow-through）
- 夸张但不失真

**关键动画：**

**待机动画：**
- 轻微呼吸
- 偶尔动作（整理装备、环顾四周）
- 循环流畅

**移动动画：**
- 行走：6帧，慢速
- 跑步：6帧，快速
- 脚步与地面接触感

**战斗动画：**
- 攻击：明显的前摇和后摇
- 闪避：快速移动，残影效果
- 受伤：击退效果，无敌帧
- 死亡：倒地动画，粒子效果

### 特效动画

**攻击特效：**
- 近战：挥砍轨迹、冲击波
- 远程：子弹轨迹、爆炸
- 技能：独特的视觉效果

**环境特效：**
- 火焰：跳动的火光
- 水面：波纹效果
- 风：草木摇摆
- 雨雪：粒子系统

**UI动画：**
- 淡入淡出
- 滑动效果
- 弹出动画
- 数字跳动

## 光照和阴影

### 光照系统

**全局光照：**
- 白天：明亮的自然光
- 夜晚：月光和人造光源

**动态光源：**
- 火把：橙黄色，闪烁
- 路灯：白色，稳定
- 窗户：暖黄色
- 特殊光源：根据场景

**光照范围：**
- 玩家视野：圆形光照
- 光源衰减：自然过渡
- 阴影：简单的投影

### 夜晚视野系统

**视野限制：**
- 黑暗区域：完全不可见
- 光照区域：正常可见
- 过渡区域：半透明

**实现方式：**
- 使用 Unity 2D Lights
- 或自定义 Shader
- 雾效（Fog of War）

## 美术资源规范

### 文件命名规范

**格式：**
```
[类型]_[名称]_[变体]_[帧数].png

示例:
char_player_idle_01.png
char_player_walk_01.png
enemy_zombie_attack_01.png
item_potion_health.png
tile_ground_dirt_01.png
ui_button_normal.png
```

### 文件组织

```
Art/
├── Characters/
│   ├── Player/
│   │   ├── Idle/
│   │   ├── Walk/
│   │   └── Attack/
│   ├── NPCs/
│   └── Enemies/

    ├── Combat/
```

### 导出设置

**Sprite设置：**
- Pixels Per Unit：16（或32）
- Filter Mode：Point（无过滤）
- Compression：None（保持像素清晰）
- Max Size：根据实际需要

**动画设置：**
- Frame Rate：12 FPS（像素风格标准）
- Loop：根据动画类型
- Sprite Sheet：使用 Unity Sprite Editor 切割

## 美术制作流程

### 概念设计阶段

1. **草图绘制**
   - 手绘或数字草图
   - 确定基本形状和比例
   - 多个方案对比

2. **色彩方案**
   - 确定调色板
   - 测试不同配色
   - 考虑色盲友好

3. **风格测试**
   - 制作样本
   - 测试在游戏中的效果
   - 调整细节

### 制作阶段

1. **线稿**
   - 清晰的轮廓
   - 确定像素位置
   - 保持一致性

2. **上色**
   - 基础色
   - 阴影和高光
   - 细节添加

3. **动画**
   - 关键帧
   - 中间帧
   - 循环测试

4. **导出**
   - 检查尺寸
   - 优化文件大小
   - 导入Unity测试

### 迭代优化

1. **游戏内测试**
   - 实际运行效果
   - 可读性检查
   - 性能测试

2. **反馈收集**
   - 团队评审
   - 玩家测试
   - 数据分析

3. **调整优化**
   - 修改问题
   - 提升质量
   - 保持一致性

## 性能优化

### 美术优化策略

**Sprite Atlas：**
- 将相关精灵打包
- 减少 Draw Call
- 提高渲染效率

**动画优化：**
- 共享动画控制器
- 减少不必要的帧
- 使用对象池

**粒子优化：**
- 限制粒子数量
- 使用简单的粒子
- 及时销毁

**光照优化：**
- 限制动态光源数量
- 使用烘焙光照
- 简化阴影计算

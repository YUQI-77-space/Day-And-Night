# 敌人动画资源需求文档

> **文档版本：** v1.0  
> **创建日期：** 2026-02-13  
> **适用阶段：** Alpha阶段  
> **负责部门：** 美术部

---

## 文档概述

本文档详细说明游戏中敌人的动画资源需求，包括各类敌人的Idle、Walk、Attack、Hurt、Die等基础动画，以及特殊技能动画和Boss阶段变化动画。动画是敌人表现力的核心，需要传达出末日敌人的威胁感和独特特征。

---

## 动画基础规范

### 动画标准

| 项目 | 规格 | 说明 |
|------|------|------|
| 帧率 | 12 FPS | 像素风格标准帧率 |
| 循环类型 | 根据动画类型 | Idle/Walk循环，Attack单次 |
| 方向 | 四方向 | 上、下、左、右 |
| 角色尺寸 | 32×32像素 | 与静态资源一致 |
| 输出格式 | PNG序列帧 | 或Sprite Sheet格式 |

### 动画帧数标准

| 动画类型 | 基础帧数 | 循环方式 | 敌人类型 |
|----------|----------|----------|----------|
| Idle（待机） | 4-6帧 | 循环 | 所有敌人 |
| Walk（移动） | 6-8帧 | 循环 | 移动型敌人 |
| Run（奔跑） | 6-8帧 | 循环 | 快速型敌人 |
| Attack（攻击） | 4-8帧 | 单次 | 所有敌人 |
| Special（特殊技能） | 8-16帧 | 单次 | 精英/Boss |
| Hurt（受伤） | 2-4帧 | 单次 | 所有敌人 |
| Die（死亡） | 6-8帧 | 单次 | 所有敌人 |
| Spawn（生成） | 4-6帧 | 单次 | 所有敌人 |
| Despawn（消失） | 4帧 | 单次 | 召唤型敌人 |

## 基础敌人动画

### 史莱姆系列动画

#### 普通史莱姆动画

| 动画类型 | 帧数 | 循环 | 说明 |
|----------|------|------|------|
| Idle | 4帧 | ✅ | 果冻般轻微晃动 |
| Walk | 6帧 | ✅ | 弹跳前进，身体伸缩 |
| Attack | 6帧 | ❌ | 蓄力弹跳攻击 |
| Hurt | 2帧 | ❌ | 身体收缩变形 |
| Die | 6帧 | ❌ | 瘫平、消散 |

#### Idle动画帧内容

```
方向：下（South）- 4帧循环

帧1：基础形状，略微扁平
帧2：向上拉伸，变得更圆
帧3：恢复基础形状
帧4：向下收缩，微微变扁
帧5-重复帧1
```

#### Walk动画帧内容

```
方向：下（South）- 6帧循环

帧1：起始位置，身体扁平
帧2：开始向上弹起
帧3：空中最高点，身体最圆
帧4：下落中，开始变形
帧5：落地，略微压扁
帧6：恢复起始形状，准备下一次弹跳
```

#### Attack动画帧内容

```
方向：下（South）- 6帧

帧1-2（准备）：身体收缩，蓄力
帧3-4（攻击）：弹起，快速扑向目标
帧5（命中）：身体展开，接触目标
帧6（收招）：缓慢恢复原状
```

#### Die动画帧内容

```
6帧

帧1：受到致命伤害，身体剧烈收缩
帧2：开始瘫平
帧3：进一步瘫平，边缘模糊
帧4：颜色变淡
帧5：几乎透明
帧6：完全消失
```

### 丧尸系列动画

#### 普通丧尸动画

| 动画类型 | 帧数 | 循环 | 说明 |
|----------|------|------|------|
| Idle | 4帧 | ✅ | 身体前后摇摆 |
| Walk | 6帧 | ✅ | 蹒跚行走，步伐沉重 |
| Attack | 6帧 | ❌ | 抓挠、撕咬动作 |
| Hurt | 2帧 | ❌ | 后仰、惨叫 |
| Die | 8帧 | ❌ | 倒地、抽搐、静止 |

#### Idle动画帧内容

```
方向：下（South）- 4帧循环

帧1：站立姿势，身体略微前倾
帧2：身体重心后移，头部后仰
帧3：恢复帧1姿势
帧4：身体前倾，肩膀耸动
```

#### Walk动画帧内容

```
方向：下（South）- 6帧循环

帧1：右脚抬起，身体重心在左腿
帧2：右脚向前迈出，重心前移
帧3：右脚落地，身体前倾
帧4：左脚抬起
帧5：左脚向前迈出
帧6：左脚落地，身体略微后仰
```

#### Attack动画帧内容

```
方向：下（South）- 6帧

帧1-2（准备）：身体后仰，手臂后摆
帧3（攻击）：手臂向前抓挠，身体前扑
帧4（命中）：爪子接触目标
帧5-6（收招）：缓慢收回手臂
```

#### Die动画帧内容

```
8帧

帧1：受到致命伤害，身体后仰
帧2：膝盖弯曲
帧3：单膝跪地
帧4：完全倒地
帧5-6：身体抽搐
帧7：停止抽搐
帧8：完全静止
```

## Boss动画设计

### Boss动画特点

| 特征 | 说明 |
|------|------|
| 尺寸 | 64×64像素或更大 |
| 阶段动画 | 多阶段变形动画 |
| 大招动画 | 特殊技能完整动画 |
| 召唤动画 | 召唤小怪的过程 |
| 死亡动画 | 壮观的死亡演出 |

### 阶段变化动画

#### 阶段1 → 阶段2 变化

```
触发条件：HP降至75%

帧1-2：Boss受到伤害
帧3-4：身体开始发光
帧5-6：身体变形
帧7-8：新形态出现
帧9-10：完成变化，姿态摆好
帧11-12：新形态Idle动画
```

#### 阶段2 → 阶段3 变化

```
触发条件：HP降至50%

帧1-4：发出怒吼
帧5-8：身体膨胀
帧9-12：释放能量
帧13-16：狂暴化完成
帧17-20：狂暴形态Idle
```

## 动画状态机

### 敌人状态转换

```
                    ┌─────────────────┐
                    │     Spawn       │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │      Idle       │◄────────────────┐
                    └────────┬────────┘                 │
                             │                          │
              ┌──────────────┼──────────────┐           │
              │              │              │           │
     ┌────────▼─────┐  ┌────▼────┐   ┌────▼────┐       │
     │    Walk      │  │  Run    │   │Special  │       │
     └──────┬───────┘  └────┬───┘   └────┬────┘       │
            │               │            │             │
            └───────────────┴────────────┘             │
                             │                          │
                    ┌────────▼────────┐                 │
                    │    Attack      │─────────────────┤
                    └────────┬────────┘                 │
                             │                          │
                    ┌────────▼────────┐                 │
                    │    Hurt        │─────────────────┤
                    └────────┬────────┘                 │
                             │                          │
              ┌──────────────┼──────────────┐           │
              │              │              │           │
     ┌────────▼─────┐  ┌────▼────┐   ┌────▼────┐       │
     │   Idle Hurt  │  │Run Hurt │   │   Die   │       │
     └──────────────┘  └────────┘   └─────────┘       │
```

### 优先级规则

1. **最高**：Die > Special > Attack > Spawn
2. **高**：Hurt
3. **中**：Run > Walk
4. **低**：Idle

## Sprite Sheet格式

### 排列方式

推荐使用Sprite Sheet打包敌人动画：

```
# 基础敌人Sprite Sheet (4方向×6帧)
[Idle_Down_1]...[Idle_Down_4]
[Idle_Up_1]...  [Idle_Up_4]
[Idle_L_1]...   [Idle_L_4]
[Idle_R_1]...   [Idle_R_4]

[Walk_Down_1]...[Walk_Down_6]
[Walk_Up_1]...  [Walk_Up_6]
[Walk_L_1]...   [Walk_L_6]
[Walk_R_1]...   [Walk_R_6]
```

### 导出设置

| 设置项 | 推荐值 |
|--------|--------|
| 网格大小 | 32×32像素（精英48×48，boss64×64） |
| 间距 | 0像素 |
| 格式 | PNG-8或PNG-32 |
| 压缩 | 无（保持像素清晰） |

## 文件命名规范

```
# 基础敌人动画
enemy_[名称]_[动画]_[方向]_[帧数].png
示例：
enemy_slime_idle_down_4f.png
enemy_zombie_walk_down_6f.png

# 精英敌人动画
enemy_elite_[名称]_[动画]_[帧数].png
示例：
enemy_elite_fire_slime_attack_10f.png

# Boss动画
enemy_boss_[名称]_[动画]_[帧数].png
示例：
enemy_boss_infested_general_idle_6f.png

# Boss阶段动画
enemy_boss_[名称]_phase_[阶段]_[帧数].png
示例：
enemy_boss_infested_general_phase2_20f.png

# 特效动画
vfx_[特效类型]_[编号]_[帧数].png
示例：
vfx_hit_slash_4f.png
vfx_death_explosion_8f.png

# 状态动画叠加
status_[状态类型]_[帧数].png
示例：
status_burning_6f.png
status_frozen_6f.png
```

### 动画原则

1. **威胁感**：动画要传达敌人的危险性
2. **可读性**：攻击动作要有前摇
3. **打击感**：受伤和死亡要有反馈
4. **流畅性**：循环动画要自然
5. **差异化**：不同敌人有独特动画

## 注意事项

1. **一致性**：所有敌人动画风格统一
2. **方向处理**：左/右方向建议镜像
3. **性能考虑**：控制特效粒子数量
4. **测试验证**：在游戏中测试动画效果
5. **打击感**：注重攻击和受伤的反馈

# 主角动画资源需求文档

> **文档版本：** v1.0  
> **创建日期：** 2026-02-13  
> **适用阶段：** Alpha阶段  
> **负责部门：** 美术部

## 文档概述

本文档详细说明主角（玩家角色）的动画资源需求，包括所有动作状态、动画帧数规范、方向处理方式以及各性别的动画配置。动画是角色表现力的核心，需要在有限的像素空间中传达清晰的动作信息。

---

## 基础规格

### 动画标准

| 项目 | 规格 | 说明 |
|------|------|------|
| 帧率 | 12 FPS | 像素风格标准帧率 |
| 循环类型 | 根据动画类型 | 行走/待机循环，攻击单次 |
| 方向 | 四方向 | 上、下、左、右 |
| 角色尺寸 | 32×32像素 | 与静态资源保持一致 |
| 输出格式 | PNG序列帧 | 或Sprite Sheet格式 |

### 动画帧数标准

| 动画类型 | 基础帧数 | 循环方式 | 用途说明 |
|----------|----------|----------|----------|
| Idle（待机） | 4-6帧 | 循环 | 静止状态 |
| Walk（行走） | 6-8帧 | 循环 | 慢速移动 |
| Run（奔跑） | 6-8帧 | 循环 | 快速移动 |
| Attack（攻击） | 4-8帧 | 单次 | 近战/远程攻击 |
| Dodge（闪避） | 4-6帧 | 单次 | 快速位移 |
| Hurt（受伤） | 2-4帧 | 单次 | 受击反馈 |
| Die（死亡） | 6-8帧 | 单次 | 死亡表现 |
| UseItem（使用道具） | 4-6帧 | 单次 | 消耗品使用 |

## Idle动画（待机状态）

### 动画说明

待机动画是角色静止时的微动作，用于表现角色的呼吸感和生命力。设计时应注意：

- **节奏**：缓慢、有规律，让玩家感到角色是"活"的
- **内容**：胸部轻微起伏、头部微动、手部自然放置
- **风格**：末日风格的角色可以有轻微颓废感，但不失生气

### 各方向动画帧数

| 方向 | 帧数 | 动画内容描述 |
|------|------|--------------|
| 下（South） | 4帧 | 正面站立，胸部轻微起伏，眼神轻微移动 |
| 上（North） | 4帧 | 背面/背面侧方，衣服轻微飘动 |
| 左（West） | 4帧 | 左侧立，左脚轻微重心转移 |
| 右（East） | 4帧 | 右侧立，右脚轻微重心转移 |

### 帧内容详细说明

```
方向：下（South）

帧1：基础站立姿势，双手自然下垂
帧2：胸部轻微抬起（吸气）
帧3：恢复基础姿势
帧4：胸部轻微下降（呼气），眼睛轻微眨动
```

```
方向：上（North）

帧1：背面站立，背包/披风自然垂下
帧2：身体轻微前倾
帧3：恢复基础姿势
帧4：身体轻微后仰，披风轻微飘动
```

```
方向：左/右（West/East）

帧1：侧面站立，重心居中
帧2：重心转移到支撑腿
帧3：恢复基础姿势
帧4：重心略微偏移
```

### 特殊待机变体

| 变体名称 | 帧数 | 使用场景 | 备注 |
|----------|------|----------|------|
| 战斗待机 | 4帧 | 战斗状态中 | 双手持武器，身体略微前倾 |
| 交谈待机 | 4帧 | 与NPC对话时 | 双手自然站立，面朝对话对象 |
| 休息待机 | 4帧 | 室内安全区域 | 更加放松的姿态 |

## Walk动画（行走状态）

### 动画说明

行走是游戏中最频繁出现的动画，需要在流畅性和辨识度之间取得平衡。

- **节奏**：中等速度，有明确的脚步节奏
- **内容**：腿部交替移动、手臂自然摆动、身体轻微起伏
- **特点**：末日后期的角色行走略显疲惫，但仍保持警觉

### 各方向动画帧数

| 方向 | 帧数 | 动画内容描述 |
|------|------|--------------|
| 下（South） | 6帧 | 正面行走，双手交替摆动 |
| 上（North） | 6帧 | 背面行走，背包/披风轻微晃动 |
| 左（West） | 6帧 | 左侧行走，左脚起步 |
| 右（East） | 6帧 | 右侧行走，右脚起步 |

### 帧内容详细说明

```
方向：下（South）- 6帧循环

帧1：起步，右脚在后，左脚在前，重心居中
帧2：右脚抬起，身体前倾
帧3：右脚落地，身体前倾达到最大
帧4：左脚抬起，身体后仰
帧5：左脚落地，重心居中
帧6：完成一步循环，准备下一轮
```

```
方向：上（North）- 6帧循环

帧1：起步，背面视角，背包轻微在后
帧2：背包向前摆动
帧3：背包摆动到前方
帧4：背包开始向后摆动
帧5：背包摆动到后方
帧6：完成循环
```

### 行走方向处理

由于是俯视角，像素角色的行走方向处理需要特别注意：

- **左/右对称**：左手左脚动作对称（镜像处理）
- **下方向**：角色正面，双手摆动方向相反
- **上方向**：角色背面，双手摆动方向相反

## Run动画（奔跑状态）

### 动画说明

奔跑是比行走更快的移动方式，用于紧急情况或快速穿越危险区域。

- **节奏**：快速，有明显的冲刺感
- **内容**：腿部大幅摆动、手臂高频率摆动、身体前倾
- **特点**：更加紧张、紧迫的动作表现

### 各方向动画帧数

| 方向 | 帧数 | 动画内容描述 |
|------|------|--------------|
| 下（South） | 8帧 | 正面奔跑，双手大幅摆动 |
| 上（North） | 8帧 | 背面奔跑，披风向后飘扬 |
| 左（West） | 8帧 | 左侧奔跑，身体前倾 |
| 右（East） | 8帧 | 右侧奔跑，身体前倾 |

### 奔跑与行走对比

| 特征 | 行走 | 奔跑 |
|------|------|------|
| 帧数 | 6帧 | 8帧 |
| 身体倾斜 | 轻微 | 明显前倾 |
| 手臂摆动 | 自然摆动 | 大幅摆动 |
| 脚部抬起 | 适中 | 较高 |
| 尘土效果 | 无 | 可能需要 |

## Attack动画（攻击状态）

### 动画说明

攻击动画是最重要的战斗反馈，需要清晰传达攻击动作和时机。

- **节奏**：有明确的前摇、攻击、收招三个阶段
- **内容**：蓄力（可选）、武器挥动、命中判定、收招
- **特点**：攻击要有"力度感"，让玩家能预判攻击时机

### 近战攻击动画

#### 基础攻击（剑/刀）

| 方向 | 帧数 | 动画阶段说明 |
|------|------|--------------|
| 下（South） | 6帧 | 举刀(1)→下劈(2)→命中(3)→收刀(4)→待机(5-6) |
| 上（North） | 6帧 | 举刀(1)→上挑(2)→命中(3)→收刀(4)→待机(5-6) |
| 左（West） | 6帧 | 举刀(1)→左砍(2)→命中(3)→收刀(4)→待机(5-6) |
| 右（East） | 6帧 | 举刀(1)→右砍(2)→命中(3)→收刀(4)→待机(5-6) |

#### 攻击阶段详细说明

```
帧1-2（前摇）：角色准备攻击，举起武器
  - 动作：蓄力、瞄准
  - 目的：给玩家反应时间

帧3-4（攻击）：武器挥出
  - 动作：快速挥动武器
  - 目的：造成伤害的判定帧

帧5-6（收招）：角色恢复站立
  - 动作：收刀/收招
  - 目的：攻击后的硬直时间
```

#### 特殊攻击变体

| 攻击类型 | 帧数 | 特点 | 适用武器 |
|----------|------|------|----------|
| 重攻击 | 8帧 | 更大动作幅度，更高伤害 | 大剑、锤子 |
| 快速攻击 | 4帧 | 快速挥动，低伤害 | 匕首、小刀 |
| 范围攻击 | 10帧 | 360度旋转或横扫 | 长柄武器 |
| 连击-第一击 | 4帧 | 起手式 | 所有武器 |
| 连击-第二击 | 4帧 | 追击式 | 所有武器 |
| 连击-终结击 | 6帧 | 终结技，高伤害 | 所有武器 |

### 远程攻击动画

#### 射击（枪械）

| 方向 | 帧数 | 动画阶段说明 |
|------|------|--------------|
| 下（South） | 6帧 | 举枪(1)→瞄准(2)→射击(3)→后坐(4)→恢复(5-6) |
| 上（North） | 6帧 | 举枪(1)→射击(2)→后坐(3)→恢复(4-6) |
| 左/右（West/East） | 6帧 | 举枪(1)→射击(2)→后坐(3)→恢复(4-6) |

#### 法杖攻击（魔法）

| 方向 | 帧数 | 动画阶段说明 |
|------|------|--------------|
| 下（South） | 6帧 | 举杖(1)→凝聚(2)→释放(3)→光芒(4)→恢复(5-6) |
| 上（North） | 6帧 | 举杖(1)→凝聚(2)→释放(3)→光芒(4)→恢复(5-6) |
| 左/右（West/East） | 6帧 | 举杖(1)→凝聚(2)→释放(3)→光芒(4)→恢复(5-6) |

## Dodge动画（闪避状态）

### 动画说明

闪避是重要的生存技能，动画需要表现快速位移和无敌帧。

- **节奏**：极快，有明显的瞬间移动感
- **内容**：翻滚、快速冲刺、向后跳跃
- **特点**：位移明显，但要在有限帧数内完成

### 各方向闪避动画

| 闪避类型 | 帧数 | 适用场景 | 特点 |
|----------|------|----------|------|
| 翻滚闪避 | 6帧 | 向前方闪避 | 侧身翻滚 |
| 后跳闪避 | 4帧 | 向后闪避 | 快速向后跳跃 |
| 侧移闪避 | 4帧 | 向左右闪避 | 快速横向移动 |
| 格挡闪避 | 4帧 | 格挡成功后 | 架招防御 |

### 闪避帧内容（翻滚为例）

```
方向：下（South）- 6帧

帧1：准备闪避，身体下蹲
帧2：开始翻滚，侧身躺地
帧3：翻滚中，身体卷曲
帧4：翻滚结束，身体展开
帧5：恢复站立姿势
帧6：完全恢复，可以行动
```

### 闪避特效需求

| 特效类型 | 规格 | 说明 |
|----------|------|------|
| 残影 | 32×32，2-3帧 | 闪避时的残影效果 |
| 尘土 | 32×32，4帧 | 闪避起点的尘土 |
| 光效 | 32×32，6帧 | 无敌帧时的光圈 |

## Hurt动画（受伤状态）

### 动画说明

受伤动画是重要的反馈，让玩家知道角色受到了伤害。

- **节奏**：快速、短暂
- **内容**：向后仰、闪烁、表情变化
- **特点**：受伤要"痛"，但不能过长影响操作

### 受伤动画配置

| 受伤类型 | 帧数 | 动画描述 | 使用频率 |
|----------|------|----------|----------|
| 轻微受伤 | 2帧 | 身体轻微后仰 | 常见 |
| 中度受伤 | 4帧 | 明显后仰，可能跪地 | 较少 |
| 重度受伤 | 4帧 | 击飞效果 | 很少 |

### 受伤帧内容

```
帧1：受到攻击瞬间，身体向后仰
帧2：达到最大后仰角度，受伤表情
帧3：开始恢复直立
帧4：完全恢复站立
```

### 受伤颜色特效

| 效果类型 | 规格 | 颜色 | 持续时间 |
|----------|------|------|----------|
| 受伤闪烁 | 全屏或角色 | 红色半透明 | 0.2秒 |
| 受伤边框 | 角色周围 | 红色 | 0.5秒 |
| 受伤屏幕效果 | 全屏 | 红色暗角 | 0.3秒 |

## Die动画（死亡状态）

### 动画说明

死亡动画是角色的最后表现，需要有足够的戏剧性和完成感。

- **节奏**：中等速度，有明确的死亡过程
- **内容**：倒地、消失、粒子效果
- **特点**：末日风格的死亡可以带有悲壮感

### 死亡动画配置

| 死亡类型 | 帧数 | 动画描述 | 后续效果 |
|----------|------|----------|----------|
| 普通死亡 | 6帧 | 缓慢倒地，身体化为灰烬 | 粒子消散 |
| 痛苦死亡 | 8帧 | 挣扎后倒地 | 血液粒子 |
| 英雄死亡 | 8帧 | 站立倒下，有定格画面 | 慢动作 |

### 死亡帧内容

```
普通死亡 - 6帧

帧1：受到致命伤害，表情震惊
帧2：身体开始摇晃
帧3：单膝跪地
帧4：完全倒地
帧5：身体开始发光/消散
帧6：完全消失
```

## UseItem动画（使用道具）

### 动画说明

使用道具时的动作反馈，让玩家确认道具已被使用。

- **节奏**：中等，明确的使用动作
- **内容**：拿出道具、使用、恢复
- **特点**：要有明确的使用动作，让玩家确认

### 常见道具使用动画

| 道具类型 | 帧数 | 动画描述 | 备注 |
|----------|------|----------|------|
| 生命药水 | 6帧 | 拿出→喝下→恢复 | 喝药动作 |
| 状态药剂 | 6帧 | 拿出→喝下→变色 | 颜色变化 |
| 投掷物 | 6帧 | 拿出→瞄准→投掷 | 投掷动作 |
| 使用卷轴 | 6帧 | 拿出→阅读→发光 | 阅读动作 |

## 动画状态机

### 状态转换关系

```
                    ┌─────────────────┐
                    │     Start       │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │      Idle       │◄────────────────┐
                    └────────┬────────┘                 │
                             │                          │
              ┌──────────────┼──────────────┐           │
              │              │              │           │
     ┌────────▼─────┐  ┌────▼────┐   ┌────▼────┐       │
     │    Walk      │  │  Run    │   │UseItem  │       │
     └──────┬───────┘  └────┬───┘   └────┬────┘       │
            │               │            │             │
            └───────────────┴────────────┘             │
                             │                          │
                    ┌────────▼────────┐                 │
                    │    Attack      │─────────────────┤
                    └────────┬────────┘                 │
                             │                          │
                    ┌────────▼────────┐                 │
                    │    Dodge        │─────────────────┤
                    └────────┬────────┘                 │
                             │                          │
              ┌──────────────┼──────────────┐           │
              │              │              │           │
     ┌────────▼─────┐  ┌────▼────┐   ┌────▼────┐       │
     │   Idle Hurt  │  │Run Hurt │   │Die      │       │
     └──────────────┘  └────────┘   └─────────┘       │
```

### 优先级规则

1. **最高**：Die（死亡）> Attack（攻击）> UseItem（使用道具）
2. **高**：Dodge（闪避）
3. **中**：Hurt（受伤）
4. **低**：Run（奔跑）> Walk（行走）
5. **最低**：Idle（待机）

### 动画原则

1. **预备动作**：攻击前要有明显的准备动作
2. **跟随动作**：攻击后要有收招动作
3. **节奏感**：快-慢-快的基本节奏
4. **反馈明确**：让玩家清楚知道动作是否完成

## 注意事项

1. **方向一致性**：左/右方向建议镜像处理以节省资源
2. **性别差异**：男女主角动画可以共用核心动作，但需要重新绘制
3. **武器兼容性**：不同武器可能需要不同的手持动画
4. **测试验证**：所有动画需要在游戏中实际测试手感
5. **性能考虑**：过多的粒子效果可能影响性能

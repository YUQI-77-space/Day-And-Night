## 文档概述

本文档详细说明游戏中NPC（非玩家角色）的美术资源需求，包括核心NPC、重要NPC和普通NPC的设计规范。NPC是小镇生态的重要组成部分，每个NPC都需要有独特的外观和性格特征。

---

## NPC分类体系

### NPC分类标准

| 分类等级 | 优先级 | 动画需求 | 详细程度 | 数量上限 |
|----------|--------|----------|----------|----------|
| 核心NPC | P0 | 完整动画集 | 极高 | 5-8人 |
| 重要NPC | P1 | 基础动画 | 高 | 15-20人 |
| 普通NPC | P2 | 简单动画/静态 | 中 | 不限 |
| 背景NPC | P3 | 无动画 | 低 | 不限 |

### 分类标准说明

**核心NPC（Core NPCs）**
- 推动主线剧情的关键角色
- 有完整的背景故事和个人任务
- 玩家会频繁与其互动
- 需要完整的动画集和表情系统

**重要NPC（Important NPCs）**
- 支线任务的关键角色
- 有独特的职业特征
- 玩家会定期访问
- 需要基础动画集

**普通NPC（Common NPCs）**
- 填充小镇氛围的居民
- 有简单的对话内容
- 偶发性互动
- 需要简单动画或静态外观

**背景NPC（Background NPCs）**
- 仅作为环境装饰
- 无互动功能
- 远景使用
- 可使用模块化组件

- **个性表达**：通过表情和姿态体现性格

## 文件命名规范

```
# 核心NPC
npc_core_[NPC ID]_[状态].png           # 基础外观
npc_core_[NPC ID]_idle_[方向]_[帧数].png
npc_core_[NPC ID]_walk_[方向]_[帧数].png
npc_core_[NPC ID]_special_[动作ID]_[帧数].png
npc_core_[NPC ID]_face_[表情ID].png    # 表情

# 重要NPC
npc_important_[NPC ID]_[状态].png
npc_important_[NPC ID]_idle_[帧数].png
npc_important_[NPC ID]_walk_[帧数].png

# 普通NPC
npc_common_[NPC ID]_[状态].png

# 模块化组件
npc_body_[体型ID]_[性别ID].png         # 身体基础
npc_cloth_[服装ID]_[颜色ID].png        # 服装层
npc_hair_[发型ID]_[颜色ID].png         # 发型层
npc_face_[面部ID].png                   # 面部特征
npc_accessory_[配件ID].png              # 配件层

# 示例
npc_core_n01_idle_down_4f.png          # 镇长待机下方向第4帧
npc_core_n03_walk_left_3f.png          # 铁匠行走左方向第3帧
npc_body_type01_male.png               # 男性1号体型
npc_cloth_civil_03_blue.png            # 平民装3号蓝色版
```

## 注意事项

1. **资源复用**：普通NPC尽量使用模块化组件
2. **核心NPC优先**：确保核心NPC质量最高
3. **一致性**：所有NPC动画风格保持统一
4. **可扩展性**：为未来可能增加的NPC预留空间
5. **性能优化**：大量NPC时使用Sprite Atlas

> **文档版本：** v1.0

> **负责部门：** 美术部

## 文档概述

本文档详细说明游戏中场景的美术资源需求，包括地形、环境元素、天气效果等前期最小化场景所需的基础美术资源。场景是游戏视觉体验的重要组成部分，需要在像素风格中营造出末日小镇的氛围。

## 文件命名规范

```
# 瓦片资源
tile_[类型]_[名称]_[变体].png
示例：
tile_grass_base.png
tile_grass_edge_n.png
tile_dirt_path1.png
tile_wood_floor.png
tile_wall_wood_base.png

# 环境元素
prop_[类型]_[名称]_[变体].png
示例：
prop_tree_oak_1.png
prop_rock_large.png
prop_fence_wood.png
prop_lamp_post.png

# 装饰物
decor_[类型]_[名称]_[变体].png
示例：
decor_flower_red.png
decor_grass_tall.png

# 天气特效
weather_[天气类型].png        # 粒子纹理
vfx_[特效名称].png             # 特效元素

# 动画帧
anim_[名称]_[帧编号].png       # 动态环境元素
```

### 设计原则

1. **可拼接性**：瓦片需要支持自动拼接
2. **层次感**：前景、中景、背景的区分
3. **辨识度**：不同地形一眼可辨
4. **氛围统一**：保持末日小镇的整体调性

---

## 审核标准

### 瓦片标准

- [ ] 尺寸一致（16×16或指定尺寸）
- [ ] 边缘完美拼接
- [ ] 颜色过渡自然
- [ ] 符合像素风格规范

### 环境元素标准

- [ ] 风格与主角/NPC统一
- [ ] 远近距离表现清晰
- [ ] 碰撞体积明确
- [ ] 可正确放置在Tilemap上

### 天气效果标准

- [ ] 粒子运动自然
- [ ] 性能消耗合理
- [ ] 与场景光照配合
- [ ] 符合游戏氛围

---

## 注意事项

1. **模块化设计**：尽量使用可复用的瓦片和组件
2. **性能优化**：减少大尺寸单张图片
3. **一致性**：所有场景资源保持风格统一
4. **可扩展性**：为未来场景预留资源空间
5. **测试验证**：在Unity中实际测试场景效果
